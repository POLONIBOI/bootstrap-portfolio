Task Manager

A full-stack task management application with user authentication, CRUD operations, task categories, and deadline tracking — built with Python and Django.

Overview

This Task Manager was developed as a full-stack project to deepen my understanding of back-end development with Python and the Django framework. The application allows users to register, log in, and manage their tasks through a clean, intuitive interface. Each user has a private workspace where they can create, read, update, and delete tasks.

Tasks can be organised into categories such as Work, Personal, and Study. Each task supports a priority level (Low, Medium, High), a due date, and a completion status. The project uses Django's built-in authentication system for secure user management and leverages Django's ORM for database operations with SQLite.

Architecture

The application follows Django's Model-View-Template (MVT) architecture. The Task model defines fields for title, description, category, priority, due date, and completion status — each linked to a user via a foreign key. Class-based views handle list, create, update, and delete operations with built-in permission checks to ensure users can only access their own tasks.

Django's template engine renders the front-end with Bootstrap for styling. Forms use Django's form validation for server-side input checking, and the authentication middleware secures all task-related routes.
